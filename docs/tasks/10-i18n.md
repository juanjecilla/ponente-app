# Task 10: i18n (EN/ES)

**Phase:** 6
**Estimated Effort:** 2 hours
**Dependencies:** 01 (i18n bootstrap), 06 (tag labels)

---

## Context

i18n is wired from day 1 (task 01). This task completes EN/ES coverage of **UI strings only** — speaker-entered content (name, bio, links) stays as entered. Tag labels come translated from the `tags` collection. A language switcher lets users toggle, gated by `enable_es_locale`.

## Goal

Complete `en.json`/`es.json`, a `<LanguageSwitcher>`, verified tag-label coverage, and no hardcoded UI strings.

---

## Implementation Steps

### 10.1 Strings
- Audit every component; ensure all visible UI text uses `t()`. Namespaced keys: `nav`, `auth`, `profile`, `directory`, `report`, `tiers`, `contactTypes`, `gde`, `banner`, `errors`, `common`.
- Complete `src/i18n/locales/en.json` then mirror into `es.json`.
- Pluralization + interpolation for counts (e.g. result counts, char counters).

### 10.2 Tier + contact-type labels
- Translate tier labels (`free|self-covered|needs-expenses`) and contact-type labels via i18n keys (the enum values stay canonical in data; labels are presentation).

### 10.3 Switcher — `src/components/shared/LanguageSwitcher.tsx`
- EN/ES toggle; `i18n.changeLanguage`; persists via detector's localStorage cache.
- Fire `locale_changed` with `{locale}` (task 16). Hidden if `enable_es_locale` false.

### 10.4 Tag labels
- Verify `useTags().labelFor(slug, locale)` returns ES for all seeded tags; fallback EN + warn for gaps.

---

## Corner Cases & Gotchas
- **Content vs UI:** never translate speaker content; only chrome. Make this explicit in `CLAUDE.md`.
- **Missing keys:** configure i18next `saveMissing`/`missingKeyHandler` in dev to catch untranslated strings; CI optional lint for missing ES keys.
- **Date/number formatting:** use `Intl`/i18next formatters; ES uses different decimal/date conventions (minor here, but be consistent).
- **Flag-off ES:** when `enable_es_locale` false, force `en` and hide switcher — but don't lose a user already on ES (fallback gracefully).
- **Layout shifts:** ES strings are ~20% longer; test buttons/labels for overflow/truncation.
- **RTL:** not needed (EN/ES both LTR) — note for future locales.

## Definition of Done
- [ ] All UI strings via `t()`; no hardcoded visible text (spot-checked + dev missing-key handler clean).
- [ ] `en.json` + `es.json` complete and key-parallel.
- [ ] `<LanguageSwitcher>` toggles + persists; respects `enable_es_locale`; fires `locale_changed`.
- [ ] All seeded tags have ES labels.
- [ ] No overflow with longer ES strings on key screens.
